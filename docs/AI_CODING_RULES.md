# AI Coding Rules

## General Rules
- Always give an implementation plan before coding.
- Do not rewrite the whole project unless there is a clear reason.
- Implement changes in small steps.
- Explain which files will be created or changed.
- Do not add new libraries unless necessary.
- Keep the folder structure consistent.
- Keep components readable and focused.
- Prefer simple solutions over clever solutions.

## Frontend Rules
- Use layouts for shared UI structure.
- Keep pages focused on page content.
- Move reusable UI into components.
- Keep API calls out of large JSX blocks.
- Use local state unless state is shared across multiple areas.
- Add loading, error, and empty states for API-driven UI.

## After Implementation
Explain:
1. What changed.
2. Why it changed.
3. Which files were affected.
4. How the data or route flow works.
5. How to manually test the feature.
6. Which fundamentals I should understand.
7. Which details are optional and do not need to be memorized.
