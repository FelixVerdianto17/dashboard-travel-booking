# Project Context

## Project Name
Travel Booking Dashboard

## Goal
Build a real-world frontend application to practice React, routing, layouts, authentication flow, API integration, form handling, loading/error/empty states, and deployment.

## Current Learning Phase
Month 1, Week 1: project setup, routing, layouts, documentation, and Git workflow.

## Tech Stack
- React
- Vite
- JavaScript
- React Router
- Tailwind CSS later if needed
- Zustand later only if global state is needed
- REST API later

## Current Scope
The app currently focuses on:
- Basic routing
- Public layout
- Dashboard layout
- Starter pages
- Clean folder structure
- Git workflow

## Coding Principles
- Keep code simple and readable.
- Do not overengineer.
- Prefer local state before global state.
- Separate routing from page UI.
- Use layouts for shared page structure.
- Do not add libraries without clear reason.
- Every feature should be manually tested.
